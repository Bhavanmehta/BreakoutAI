# TradeFrame Teardown — What to Steal for BreakoutAI

Source: live product walkthrough (tradeframe.io/pulse, /tutorials, /analyzer?ticker=TSLA) via user screenshots.
WebFetch returns only the SPA shell (client-rendered), so this teardown is built from the rendered screenshots, which are richer than any crawl.

Lens: everything below is filtered through BreakoutAI's real constraints — **single static HTML file, GitHub-Actions + static-JSON backend (no live server, no accounts, no real-time), India + US markets, and an honesty/backtest brand (measured base rates, sample sizes, Bayesian-shrunk reliability).** That last point matters: TradeFrame has beautiful presentation but shows scores/setups with **no proof behind them**. Our leapfrog is *their polish + our receipts.*

---

## 1. What TradeFrame does genuinely well

- **A guided first-run tour** — coach-mark tooltips anchored to the real nav, Back/Next, each describing a section's *job* in trader language ("Your home base", "The workbench", "Monitor your universe", "This is where you build conviction").
- **A persistent 6-step activation checklist** — a "0/6 steps" pill in the header that expands into a gamified getting-started list.
- **Section identity microcopy** — every module has a one-line value prop, not just a title.
- **A "form a daily bias" home page (Pulse)** — orients the user before they drill in: global macro strip, Fear & Greed gauge, index cards, sector heatmap, and a one-line "Led: Technology +2.7% · Lagged: Industrials −1.6%".
- **Named, described screener presets** with plain-English setup labels + tags ("Tight squeeze — breakout watch" · [Squeeze][Uptrend]).
- **A data-driven Drawdown-from-Peak module** in the analyzer (current DD, max DD w/ recovery time, avg DD, days>20%, worst-drawdowns table, underwater chart).
- **A structured Trade Decision Engine** (Strategy Type / Risk Tolerance / Time Horizon → entry/stop/targets/verdict).
- **Honest AI microcopy** — "explain likely scenarios — **not predictions**".
- **Disciplined empty states** with a single clear CTA.

## 2. Steal list — mapped to BreakoutAI (prioritized)

### TIER A — high ROI, architecture-compatible, do first
1. **Guided coach-mark tour (first visit).** Pure client-side, works in one static HTML file, zero infra. Anchor tooltips to our real sections (scan table, a signal card, base-rate panel, methodology link, India/US toggle, watchlist). Trader-voice copy per section. *This is TradeFrame's single most copyable, highest-ROI asset.*
2. **6-step activation checklist (localStorage progress pill).** Our version: (1) See today's scan (2) Open a signal card (3) Read the backtested base rate behind a pick (4) Compare India vs US (5) Add a name to your watchlist (6) Read the 60-sec methodology. Gamifies the "intimidating data tool → confident user" jump.
3. **Section-identity microcopy.** One punchy line under every section title, in trader language. Nearly free; huge clarity gain. Steal the tone verbatim, incl. the "**not predictions**" honesty framing — it's *on-brand for us and off-brand for them.*
4. **Plain-English setup label + tag chips per row.** We already classify setups — surface one human phrase ("Tight coil near 50-day, breakout watch") + tag chips. Directly in our wheelhouse.
5. **Score pill + its receipt.** TradeFrame shows a bare score (68). We show the score **with the backtested base rate next to it** — "68 · setups like this reached +1R 41% of the time (n=612)". This is the leapfrog in one component.

### TIER B — data-driven, buildable in the GitHub Action from data we already have
6. **Drawdown-from-Peak module.** Computable purely from our OHLC (`data/*/ohlc/*.json`) in the cron job → static JSON. Current DD, max DD + recovery days, avg DD, days>20%, worst-drawdowns table, underwater sparkline. Reframes risk for a breakout trader and pairs perfectly with base rates. Strong, honest, uniquely ours.
7. **Precomputed trade-plan card per pick.** We already compute entry/stop/target (+1R). Present it as TradeFrame's structured "Trade Decision" card (entry zone / stop / target / one-word verdict) — but precomputed in the scan, not a gated LLM call.
8. **"Market context" strip (our Pulse-lite).** A small daily-bias header generated by the Action: India → Nifty/BankNifty/India-VIX + FII-DII flow (we already have `HAS_FII_DII_FLOW`); US → SPY/QQQ/VIX. Add the cheap "Led / Lagged sector" one-liner. Sets context before the scan.
9. **Named "lenses" over the same scan.** Reframe our one scan as a few named presets with descriptions (e.g. "Tight-squeeze breakouts", "Pullbacks in uptrend", "High-conviction / high base-rate"). Same data, better navigability.

### TIER C — nice, lower priority
10. Persistent watchlist rail (localStorage) accessible from any view.
11. ⌘K / Ctrl-K ticker search.
12. Trending / quick-jump chips.
13. Analyst-grade stacked bar (if/where data exists — likely US-only).

## 3. Do NOT copy (traps for a static, honest, backtest-first tool)
- **Trial / quota / "1/1 remaining" / PRO gating** — we have no accounts or backend. Skip entirely.
- **"LIVE" real-time quotes.** Our data is cron-updated JSON — stay honest with "as of last scan @ HH:MM", don't fake live.
- **Live news feed** — heavy integration, not our moat.
- **Broker portfolio CSV/screenshot import** — scope creep away from the breakout mission.
- **Congressional / 13F / insider "smart money"** — a whole different data domain that dilutes our focus. Tempting, but it's not the edge.
- **Per-user accounts / 2FA / upgrade toasts** — no backend, don't imitate.

---

## 4. The one-sentence takeaway
TradeFrame is a masterclass in **onboarding and presentation** wrapped around signals with **no proof**. Take their onboarding (tour + checklist + section microcopy) almost wholesale, take the Drawdown module because we can actually compute it, and everywhere they show a number, **show ours with the backtested receipt underneath.** That's a category they structurally can't follow us into.
